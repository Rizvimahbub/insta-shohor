How does Java Script work?

Answer : JavaScript is a single-threaded programming language, it only has one Call Stack. As a result, it can only accomplish one thing at a time. The Call Stack is a data structure that keeps track of where we are in the program. We put a function on top of the stack when we step into it.The original code is run through a compiler software, which converts it to bytecode that the machine can understand and execute. JavaScript, on the other hand, does not require any compilation. Instead, a browser interpreter examines the JavaScript code, understands each line, and then executes it.



What does Java Script event loop do?

Answer : The runtime model of JavaScript is based on an event loop, which executes the code, collects and processes events, and executes queued sub-tasks. Although the language is single-threaded, the browser APIs run in their own thread. The event loop aids this process by checking if the call stack is empty on a regular basis. New functions are added from the event queue if it is empty. If it isn't, the current function call will be handled.



What is the difference between local storage and session storage?

Answer : Session Storage is similar to Local Storage, with the exception that data in Session Storage is removed after the page session ends, whereas data in Local Storage does not expire. When a document is loaded in a browser tab, a new page session is established and allocated to that tab.
